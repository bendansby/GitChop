GitChop is now open source under the MIT license. Source at github.com/bendansby/GitChop.

What's in 1.0

  • Drag-reorder commits with verb chips on every row — pick, reword, edit, squash, fixup, drop. Squash and fixup show absorb badges so chains are visible at a glance.
  • Split a commit by marking it edit: a sheet lists every hunk, you assign each to a named bucket, and each bucket becomes its own commit at apply time. The terminal-only edit → --continue dance, done as a UI.
  • Reword in place via a focused modal preloaded with the commit's full message. Applied during the rebase via a tiny $GIT_EDITOR helper — no editor pop-ups.
  • Conflict pause sheet when git stops on conflicts. Open conflicted files in your editor, click Continue. Or Skip the commit, or Abort. Mid-rebase reorder of the still-pending commits is supported from the same sheet.
  • Custom rebase base. Right-click any commit → Use as base. Equivalent to git rebase -i <sha> from the terminal.
  • Backup ref every Apply. Pre-rebase HEAD is saved to refs/gitchop-backup/<timestamp>. Failed rebases auto-rollback; manual recovery is one git update-ref away.
  • Multi-repo tabs, persistent across launches. Tab strip auto-hides when only one repo is open.
  • Preferences (⌘,) for default load depth, custom git path, and editor selection (system default, a specific .app, or a shell command like code --wait $FILE).
  • Auto-update via Sparkle.

Built for macOS

  • Pure SwiftUI. macOS 14 Sonoma or later, Apple Silicon.
  • Notarized, signed, gatekeeper-friendly.
  • No analytics, no account, no servers. The only network call is the daily Sparkle update check.
