What's new

  • Reword in place. Pick reword from a commit's verb chip — a modal opens with the full original message preloaded. Edit, save, applied at rebase time without any editor pop-ups.
  • Conflict pause. When a rebase stops on a merge conflict, GitChop now stays open with a list of unmerged files. Open each in your editor, click Continue. Or Skip the commit, or Abort the whole rebase. No bouncing back to the terminal.
  • Mid-rebase reorder. While paused on a conflict, the still-pending commits show up alongside as a drag-reorderable list. Reorder them before clicking Continue and git picks up the new sequence.
  • Custom rebase base. Right-click any commit → Use as base. Everything newer than it becomes the editable plan. Equivalent to git rebase -i <sha> from the terminal.
  • Split-bucket previews. An edit commit with a configured split now expands inline to show one ghost row per bucket — colored badge, em-dash placeholder hash, the bucket subject — so you can see at a glance what the split will produce.
  • Reset toolbar button. Discards in-flight plan edits without touching git. Confirmation prompt before discarding.
  • Chronology column. Compact relative-age token (3d, 2w, 5mo) on every row, with the absolute date as a tooltip.

Robustness

  • Split-commit re-parse. Each bucket's hunks are now re-parsed against the live working tree before applying. Splits no longer fail when an upstream rebase step has shifted the file's line numbers.
  • Stale-rebase recovery. If GitChop crashes or you force-quit mid-rebase, the next Apply detects the orphaned .git/rebase-merge and cleans it up before starting fresh.
  • Empty cherry-pick auto-skip. When resolving a conflict produces no changes (because the change is already in HEAD), GitChop runs --skip automatically instead of stalling on git's "now empty" error.

Polish

  • Drop rows get a soft red wash; bucket ghost rows are tinted to match their bucket card colors.
  • Live "Loaded N of M" status text — never goes stale relative to the actual loaded plan.
  • Verb chip menu's Edit option is grayed out with "Not available — commit has only 1 hunk" when splitting wouldn't produce multiple commits.
  • New app icon.
